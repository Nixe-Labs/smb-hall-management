# SMB Hall — Native app runbook

The web app and the native (iOS + Android) apps share one codebase.
The native shells live in `ios/` and `android/`, generated by Capacitor
and committed to git. You produce a build by syncing the web bundle
into the native shells and then opening Xcode / Android Studio to
build the actual `.ipa` / `.aab`.

This doc is the **operator runbook** — read it top to bottom the first
time, then it becomes a reference.

---

## TL;DR

```bash
# 1. Sync web → native shells
npm run cap:sync           # both platforms
# (or cap:sync:ios / cap:sync:android)

# 2. Open the native IDE to build + run
npm run cap:open:ios       # opens Xcode
npm run cap:open:android   # opens Android Studio
```

That's the whole loop. The rest of this doc is **setup** (first time)
and **submission** (when you ship to the stores).

## One-time setup

### Apple Developer

- ✅ You already have an Apple Developer account.
- Inside **Apple Developer → Certificates, Identifiers & Profiles**:
  - Register the bundle ID **`com.nixelabs.smbhall`** (must match
    `capacitor.config.ts`).
  - Provision a development certificate + distribution certificate.
- Inside **App Store Connect**:
  - Create a new app, link it to the bundle ID.
  - Fill in the metadata (description, screenshots, age rating,
    privacy policy URL).

### Local toolchain

| Tool | Why | Install |
|---|---|---|
| Xcode 15+ | iOS builds | Mac App Store |
| Xcode Command Line Tools | Required by Capacitor | `xcode-select --install` |
| Android Studio | Android builds + emulator | https://developer.android.com/studio |
| Java 17 JDK | Android Gradle | Bundled with Android Studio, or `brew install openjdk@17` |

> Capacitor 8 uses **Swift Package Manager**, so **CocoaPods is no
> longer required**. Older guides will tell you otherwise — ignore
> them.

After installing Android Studio, set the `ANDROID_HOME` env var:

```bash
# In ~/.zshrc
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator
```

Verify:

```bash
xcodebuild -version   # Xcode 15+
java -version         # 17+
adb version           # Android Debug Bridge present
```

## Day-to-day development

### Make a code change → see it in the native app

```bash
# 1. Edit any .vue / .ts file.
# 2. Sync to both shells:
npm run cap:sync
# 3. The native IDEs will pick up the change on next build.
```

Want hot-reload from the dev server instead? You can point Capacitor
at your local Vite dev server — useful for fast iteration:

```bash
# Terminal 1
npm run dev      # vite serves at http://192.168.x.x:5173

# Terminal 2 — point Capacitor at it:
# Edit capacitor.config.ts and add:
#   server: { url: 'http://192.168.x.x:5173', cleartext: true }
# Then re-sync and run.
npm run cap:sync
npm run cap:run:ios
```

Remember to **revert that server URL change before a production
build**, otherwise the shipped app will try to phone home to your
laptop.

### App icon + splash screen

Icons come from a single 1024×1024 master:

```bash
# Drop your icon at:  assets/icon-1024.png
# Drop your splash at: assets/splash-2732.png  (recommend 2732×2732, square)

# Generate every iOS + Android size:
npx capacitor-assets generate
```

`@capacitor/assets` produces all the device sizes for both platforms.

### Brand colors

Currently configured in `capacitor.config.ts`:

```ts
SplashScreen.backgroundColor = '#f4efe6'  // cream / paper
StatusBar.backgroundColor    = '#f4efe6'
```

Edit these in `capacitor.config.ts` and re-sync.

## Building for the store

### iOS — TestFlight & App Store

```bash
# 1. Build + sync the latest web bundle
npm run cap:sync:ios

# 2. Open Xcode
npm run cap:open:ios

# 3. In Xcode:
#    - Pick "Any iOS Device" as the target.
#    - Product → Archive.
#    - When the Organizer opens, "Distribute App" → "App Store Connect".
#    - It uploads to App Store Connect; processing takes ~15 minutes.
#    - In App Store Connect, the build appears under TestFlight first.
#    - Submit for review when ready.
```

First-submission Apple review: **1–3 weeks**. Subsequent updates
usually clear in 24–48 hours.

### Android — Play Console

```bash
# 1. Build + sync
npm run cap:sync:android

# 2. Open Android Studio
npm run cap:open:android

# 3. In Android Studio:
#    - Build → Generate Signed Bundle / APK → "Android App Bundle".
#    - Sign with your release keystore.
#    - Output: app-release.aab
#    - Upload to Play Console → Internal testing track.
#    - Promote to closed → open → production tracks.
```

First-submission Google review: **1–3 days**.

> **Keystore file** (`android/app/release-key.keystore` or similar)
> must NEVER be committed to git. The `.gitignore` blocks
> `keystore.properties` already. Store the keystore + its passphrase
> in your password manager — if you lose it, you cannot ship updates
> under the same Play listing, ever.

## Project layout

```
.
├── src/                    # Vue codebase (shared between web + native)
├── dist/                   # Built web bundle (regenerated each build)
├── public/                 # Static assets (favicon, PWA icons)
│
├── capacitor.config.ts     # Bundle ID, app name, plugin settings
│
├── ios/                    # Xcode project (COMMITTED)
│   └── App/
│       ├── App/            # Native iOS source
│       ├── App.xcodeproj/
│       └── Package.swift   # Capacitor 8 SPM manifest
│
└── android/                # Android Studio project (COMMITTED)
    ├── app/                # Native Android source
    ├── gradle/
    └── settings.gradle
```

`ios/App/App/public/` and `android/app/src/main/assets/public/` are
gitignored — they're regenerated from `dist/` on every sync.
